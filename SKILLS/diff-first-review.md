# Skill: Diff-First Review

Purpose
- Review changes by starting with the diff, then tracing to context as needed.

When to use
- Code review requests or PR audits.

Checklist
- Start with git diff to spot behavior changes.
- Trace to referenced contract/plan sections.
- Identify risk, regression, or missing tests.
- Summarize findings by severity with file paths.

Output
- Findings list with file references and recommended fixes/tests.
